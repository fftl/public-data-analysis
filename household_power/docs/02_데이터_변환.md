# 2단계: 데이터 변환 (Silver Layer)

## 목표

MySQL에 저장된 RAW 데이터를 읽어 품질 문제를 처리하고, 분석에 적합한 형태로 정제하여 **BigQuery Silver 테이블**(`household_power.raw`)에 적재합니다.

---

## 처리 대상 문제

수집된 원본 데이터에는 다음과 같은 품질 문제가 존재하였습니다.

### 1. 음수 값 및 이상치

전력 사용량, 평균 요금이 음수로 입력된 레코드가 존재하였습니다. 물리적으로 불가능한 값이므로 결측치(NaN)로 처리합니다.

- **이상치 사례**: 2015년 1월~2월의 일부 지역 데이터가 전국 평균 대비 약 10배 높은 값으로 수집됨
- 원인 불명으로 해당 값도 동일하게 결측치 처리

### 2. 결측치 보간

음수 변환으로 생긴 NaN은 **지역별 그룹 내에서 선형 보간(interpolation)** 으로 채웁니다.

```python
df['power_usage'] = (
    df.groupby(['sd_name', 'sgg_name'])['power_usage']
      .transform(lambda x: x.interpolate(method='linear'))
)
```

같은 지역의 인접 월 데이터를 이용해 자연스럽게 보정하는 방식입니다.

### 3. 행정구역 변경에 따른 지역명 불일치

시간이 흐르면서 행정구역 개편으로 지역명이 변경된 사례가 발생하였습니다.

**대표 사례: 부천시**

- 과거 데이터: `부천시 원미구`, `부천시 소사구`, `부천시 오정구` (3개 구 분리)
- 현재 데이터: `부천시` (구 폐지 후 단일 행정구역)

동일 기간의 통계를 내기 위해 분리된 3개 구를 **가구 수 가중평균**으로 합산하여 `부천시`로 통합합니다.

```python
weighted_avg = Σ(평균사용량 × 가구수) / Σ(가구수)
```

> 시군구(sgg_name) 기준 처리를 할 경우, 이런 불일치가 발생합니다.
> 광역시도(sd_name) 기준으로만 집계한다면 이 문제는 발생하지 않습니다.

---

## 변환 흐름 (scripts/household_power_transform.ipynb)

```
MySQL RAW 데이터 읽기
    ↓
음수 값 → NaN 변환
    ↓
지역별 그룹 내 선형 보간으로 NaN 채우기
    ↓
행정구역 변경 지역 통합 (가중평균)
    ↓
데이터 타입 검증 및 변환 (INT, FLOAT)
    ↓
BigQuery Silver 테이블 적재 (household_power.raw)
```
