# 4단계: 수집 자동화 (Airflow)

## 목표

최초 수집 이후 매월 추가되는 신규 데이터를 자동으로 감지하고 MySQL에 저장하는 **Airflow DAG**를 구성합니다.

---

## DAG 동작 흐름

```
1. DB에서 가장 최근에 수집된 연월 확인
    ↓
2. 그 다음 달 데이터가 API에 존재하는지 확인
    ↓
3. 신규 데이터가 있는 경우 → 수집 실행
   신규 데이터가 없는 경우 → 종료
    ↓
4. 17개 광역시도 순회하며 수집
    ↓
5. 실패한 지역은 fail_list에 기록 후 최대 10회 재시도
    ↓
6. 재시도 후에도 실패한 항목은 파일로 저장
```

---

### 재시도 로직

API 요청 실패 시 즉시 포기하지 않고, 실패한 지역만 모아서 재시도합니다.


최대 10회까지 재시도하며, 이후에도 실패한 항목은 파일로 기록하여 수동 처리 대상으로 남깁니다.
